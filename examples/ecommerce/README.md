# Ecommerce Example

This example shows how the v0.1 skill pack can document a checkout feature that crosses product, frontend, backend, data, and QA concerns.

Scenario: a small ecommerce shop needs a guest checkout flow with inventory reservation and payment authorization.

Documents:

- `project-brief.md`
- `prd.md`
- `trd.md`
- `ia.md`
- `user-flow.md`
- `api-spec.md`
- `erd.md`
- `qa-checklist.md`
- `doc-audit-report.md`

Use this folder to see how Dreampia documents capture cross-document dependencies such as acceptance criteria, API contracts, inventory rules, payment states, and QA coverage.
